Direction A
Intelligence First — fit_reasoning Above the Fold
The product's highest-value content is currently hidden in a collapsed accordion. Surfacing it changes the perceived value of every result.
Why this matters: The fit_reasoning field is Claude's plain-English explanation of why an investor fits this specific startup. It's the feature that turns a list into intelligence — and the thing founders screenshot and share. Currently it requires clicking "Show Fit Details." Most users never click it. Moving it above the fold is the single highest-ROI UX change in the product.

Direction B
Progressive Form — Textarea Hero, Filters Collapsed
First-time founders see 6 fields upfront. The textarea should be the entire first screen. Filters reveal only when the user wants to refine.
Why this matters: All 6 fields are currently shown on load (textarea + 3 multi-selects + budget slider). A first-time founder faces a form when they want to describe their company. The textarea is the right instinct — it's the only truly required input. Collapsing the optional filters behind "Refine search" removes cognitive load without hiding functionality that power users need.

Textarea is the hero. Filters are grouped in a single "Refine search" toggle (shown here expanded). First-time founders can submit immediately; returning users open the panel to refine. No new components needed — this is a conditional render.

Direction C
Milestone Limit — Quota as Progress, Not Punishment
The current quota error reads as a wall. Reframing it as a milestone card turns a frustrating moment into a motivating one.
Why this matters: "Monthly limit reached" is a punitive framing. Founders who hit the quota have already gotten value from the product — they've run 3 searches. That's a win. Showing "3/3 searches used this month" with a progress bar acknowledges their usage positively before presenting the upgrade path.
Current — warning banner
Monthly limit reached
You've used all 3 free searches this month. Upgrade to Pro for unlimited searches.

Amber warning banner feels like an error state. The upgrade link is inline text — easy to miss. The disabled "Find Investors" button creates a dead end.

Proposed — milestone card
3 searches completed this month
You've found investors for 3 ideas. Upgrade to Pro to keep the momentum going — unlimited searches, every month.
🎯
3 searches used 3 / 3 this month
Plans start at $29/mo · Cancel anytime

Blue milestone card acknowledges the founder's usage positively. Full-width upgrade CTA is prominent and action-oriented. Progress bar at 100% signals completion (not failure). Pricing context reduces anxiety.

Direction D
Accomplished Save — Confirming Progress Feels Real
The Save action currently swaps text labels. A micro-animation on save confirmation reinforces the "accomplished" emotional goal and makes CRM adoption feel rewarding.
Why this matters: Saving an investor is the moment a founder moves from discovery to action. The current label swap ("Save" → "Saved ✓") is functionally correct but emotionally flat. A brief scale animation + green confirmation state tells the founder: you did something real just now. This is the habit-formation trigger for SavedBoard adoption.
