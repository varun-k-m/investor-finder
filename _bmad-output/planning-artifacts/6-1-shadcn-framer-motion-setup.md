# Story S6-001 — shadcn/ui Component Library Expansion + Framer Motion Setup

**Epic:** 6 — UI Enrichment  
**Points:** 2  
**Agent:** FE Dev  
**Depends on:** None (prerequisite for all other Epic 6 stories)

---

## Goal

Install and configure all missing shadcn/ui primitives required by Epic 6, install `framer-motion` and `next-themes`, and verify the design system base is ready for subsequent stories.

---

## Acceptance Criteria

1. All listed packages install without peer-dep conflicts.
2. Each shadcn component is importable from `@/components/ui/*`.
3. `framer-motion` is importable and a simple `<motion.div>` renders without error.
4. `next-themes` `ThemeProvider` is wired into `app/layout.tsx` without breaking existing pages.
5. `tailwind.config.ts` includes the `darkMode: 'class'` setting.

---

## Implementation Tasks

### 1. Install packages

```bash
# shadcn components (run one at a time via CLI or batch-add)
npx shadcn-ui@latest add slider
npx shadcn-ui@latest add badge
npx shadcn-ui@latest add avatar
npx shadcn-ui@latest add tabs
npx shadcn-ui@latest add tooltip
npx shadcn-ui@latest add select
npx shadcn-ui@latest add command
npx shadcn-ui@latest add popover
npx shadcn-ui@latest add skeleton
npx shadcn-ui@latest add sonner

# Animation + theming
npm install framer-motion next-themes
```

### 2. `tailwind.config.ts`

Add `darkMode: 'class'` at the top level:

```typescript
const config: Config = {
  darkMode: 'class',
  // ...existing config
};
```

### 3. Wrap root layout with `ThemeProvider`

In `app/layout.tsx`, wrap `{children}` with:

```tsx
import { ThemeProvider } from 'next-themes';

// inside the <body>:
<ThemeProvider attribute="class" defaultTheme="system" enableSystem disableTransitionOnChange>
  {children}
</ThemeProvider>
```

### 4. Verify

- Import each new component in a temporary test and confirm no TS errors.
- Confirm `darkMode: 'class'` in Tailwind — adding `dark` class to `<html>` should invert colours.
